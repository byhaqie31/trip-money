## Anti-Patterns Verdict

Does this look AI-generated? **No — it passes.** The airline-diegetic system (departures roll, real-coordinate route map, perforated pass, class papers as pure token swaps) is a committed, nameable lane ("Wise-lime consumer fintech × airline ephemera") carried through every layer.

**LLM assessment**: one borderline hit — the converter pull quote's `border-l-2` left rule (TmConverter.vue) is the page's single stock pattern; neutral-ink quotation rule, defensible, but the one thing a slop-hunter could clock. Comparison strip is four structurally identical tiles always showing the first four config destinations — parallel structure is comparison semantics, but `slice(0,4)` makes it feel template rather than curated. Everything else clean: no gradient text, no hero-metric template, tracked-caps only inside the pass artifact where microprint is diegetic, zero em dashes.

**Deterministic scan**: clean — 0 findings across all app markup (exit 0). The detector did not flag the blockquote rule (neutral ink, quotation semantics); treat the LLM note as advisory.

**Visual overlays**: browser automation is unavailable in this environment, so no overlay was injected; fallback signal is the CLI scan plus source review.

## Overall Impression

A committed, warm, genuinely designed submission that reads handcrafted; 29/40 with zero P0s. The single biggest opportunity: the boarding pass is the whole point of the app, yet interaction around it leaks trust in small ways — a class tap wipes a typed budget, a shared link loses the class costume, and mobile ends on a chart instead of the pass.

## What's Working

1. **A diegetic system, not a skin** — ticker, roll, true-coordinate arcs, perforation cut from panel colour, barcode microprint, three class papers via CSS variables only. The strongest anti-slop credential the submission has.
2. **Voice + trust in the same breath** — "berapa rate hari ni?", "RM 1 buys you", the money-changer honesty line: warm register earning credibility exactly where a rate app needs it.
3. **Craft that is structural, not sprinkled** — one @theme token source, SSR-as-reduced-motion, combobox focus handling, URL-linkable passes, honest copy engineering in fmtAmount/agoLabel.

## Priority Issues

1. **[P1] Class tap destroys a typed budget** — setTravelClass always overwrites budgetSen; the control shows no amounts. Silent data loss on the core input mid-flow. Fix: show presets on segments ("Luxury · RM20k"), only overwrite when budget is still default/preset. Suggested: /impeccable polish
2. **[P1] Auto-tour rotates the whole page** — tour writes selectedCode so rate card, converter and pass all flip every 5s while reading; threshold-0 observer keeps it firing with one hero pixel visible. Fix: drive the tour through the hero's preview mechanism (already decouples display from selection) or require ≥60% visibility. Suggested: /impeccable quieter
3. **[P1] Accessibility cluster** — 13px ink/55 labels ≈3.8:1 and ink/50 meta ≈3.5:1 fail AA; First-class muted gold on champagne ≈2.1:1; ticker is 32 moving ~20px tab stops with duplicates announced; no aria-live for tour changes or "Saved ✓"; segmented controls lack pressed semantics. Fix: ink/70 labels, deepen first-class --muted, aria-hidden+tabindex=-1 the duplicate ticker run, one polite live region, sr-only "vs yesterday". Suggested: /impeccable audit
4. **[P2] Shared links drop the traveller class** — URL carries to+budget only; a shared First pass arrives as Economy ivory. The costume is the feature. Fix: add class= to URL watcher + mount parser. Suggested: /impeccable harden
5. **[P2] Hero exchange box renders a broken string pre-data** — "MYR 1 = JPY · middle rate" with a hole during load and permanently in error state; first data readout a judge sees on cold load. Fix: hide box or skeleton chip until rate exists. Suggested: /impeccable harden

## Persona Red Flags

**Casey (distracted mobile, one-handed)**: ticker chips are ~20px moving targets, effectively untappable; must scroll a 480px hero to the real picker. Tapping "Luxury" to peek at the gold pass wipes her typed RM7,500. Share copies a link instead of opening the WhatsApp share sheet, and the link arrives Economy.

**Jordan (confused first-timer)**: page changes destination by itself with no explanation or pause control ("did I do that?"); Backpacker/Traveller/Luxury don't say what they do; "▲ 0.32%" vs what and "middle rate" answered only in the footer or never.

**Sam (screen reader / keyboard)**: 32 ticker stops before main content, no skip link; ticker focus desyncs the hero headline from actual selection; "Saved ✓" not announced; pass content is div-soup, document outline is h1 → h2(leaderboard) and nothing else. The combobox itself is genuinely good once reached.

## Minor Observations

- Lime token comment promises "3 uses" but lime appears in ~5 moments; system and comment disagree.
- Compare strip vanishes entirely at budget 0 with no explanation.
- Lower sections don't render during loading, so the page grows on data arrival; skeletons don't mirror the dropdown.
- Desktop fold on 13" laptops is all mood, no product (640px hero + header).
- Hero roll leave-animation hardcodes left:12px to match pill padding; a padding change breaks it silently.
- Error card retry is ink with 10px radius where primary actions are lime with token radii; make deliberate or align.
- "Your last attempt was a moment ago" hardcoded filler in the error card.

## Questions to Consider

1. If the boarding pass is the artifact this app exists to produce, why does the mobile story end on a trend chart and a disclaimer instead of the pass?
2. Would a single scripted first-visit flight (one arc draw, one pass print, then rest) sell the map harder than an infinite loop every visitor has to kill?
3. Should traveller class ripple into content (compare strip re-ranked for an RM20k trip, per-class microprint), or is "it's just paper" the honest point?
